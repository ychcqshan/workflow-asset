# 企业资产管理系统 (EAM) 产品需求文档 (PRD)

**文档版本**: v2.0 (优化版)
**拟定人**: 系统全栈架构师

---

## 1. 文档概述及业务价值

本文档定义了企业资产管理系统 (Enterprise Asset Management, EAM) 的核心功能、业务流转及非功能需求。

**业务价值主张：**
1. **防流失与防闲置**：通过唯一身份/条码追踪，实现资产“账实相符”，降低企业资产流失率。
2. **全生命周期闭环**：从采购入库至报废核销，提供端到端的透明化管理。
3. **规范化运营**：支持公司资产的规范化管理和动态更新维护。
4. **决策支撑**：为管理层与财务提供实时数据看板，优化企业资产配置。

---

## 2. 角色与权限模型 (RBAC)

系统角色根据管理维度垂直划分，支撑不同视角的业务诉求：

*   **普通员工**：资产的使用者。负责发起资产领用、退库、报修申请，查看个人名下资产。
*   **部门领导/负责人**：审批者。负责审批本部门员工发起的资产变动申请（如领用、退库审批），包括中心主任审批和部门长审批。
*   **信息化管理员 (行政/IT)**：执行者与管理者。负责资产入库、标签生成、库存盘点、跨部门调拨、以及报修/报废的最终技术鉴定与审批。
*   **财务人员**：价值管理者。关注资产原值、折旧计算、残值及最终的财务报废核销。
*   **公司高管**：决策者。拥有全局数据穿透权限，查看公司资产看板、各部门资产总额及利用率报表。
*   **系统管理员**：配置者。负责组织架构维护、流程定义配置(BPMN设计)、数据字典扩展及系统菜单/权限分配。

## 3. 核心功能模块详细设计

### 3.1 资产台账与全生命周期管理

系统的核心是资产卡片，记录每一件单品从生到死的所有信息。

*   **资产入库（起点）**：
    *   支持手动逐条录入与基于业务模板的 Excel 批量导入。
    *   自动为每一件资产生成全局唯一标识码（资产编码），并支持生成（批量打印）。
*   **台账多维查询**：
    *   以全局视角查看资产状态列表（空闲、在用、故障、报废中、已报废等）。
    *   支持多维度高级组合检索：按资产分类、状态、所属部门、使用人、入库时间、位置点等。
*   **履历追溯（时间轴）**：
    *   在资产详情页，通过图文时间轴形式展示该资产的完整生命周期轨迹，包括何人于何时领用、归还或报修。

### 3.2 资产流转 (BPMN 引擎支撑)

所有资产状态的变更必须由标准作业流程单据驱动，实现“工作流”即“状态流”。

*   **领用流转**：
    *   流程：员工申请 -> 中心主任审批 -> 部门负责人审批 -> 信息化管理员发放。
    *   联动：审批通过并发放后，资产状态自动从“空闲”变更为“在用”，归属人变更为申请人。
*   **退库与调拨流转**：
    *   **退库**：员工离职或更换设备时发起，管理员库管确认接收后恢复为“空闲”状态。
    *   **调拨**：应对跨部门资源调度。流程需经过：转出部门审批 -> 转入部门审批 -> 管理员确认。
*   **维修与保养**：
    *   员工一键报修（支持上传现场故障图片），管理员派单或对接维修商，记录维修费用和耗时。
*   **报废核销（终点）**：
    *   资产达到使用年限或严重损坏。
    *   流程：管理员发起 -> 部门确认 -> **财务核销审批（更新财务残值）** -> 资产彻底注销，变更为“已报废”。


### 3.4 组织架构与动态审批流

*   **树状层级管理**：支持“公司-分公司-一级部门-二级中心-小组”的无限极部门树维护。
*   **结构化人事**：支持用户挂载多个部门岗位，每个部门节点强制关联“部门负责人”标识。
*   **审批流动态解析**：流程图中的审批节点配置为表达式变量，审批时由系统根据发起人的当前部门，向上动态寻址对应的负责人。

### 3.5 多维数据看板 (Dashboard)

为高管和管理员提供图形化（ECharts）报表：
*   **资产总览**：实时统计资产总数、总金额、状态分布占比（饼图）。
*   **部门资产排行**：各部门占用资产数量与金额的 Top 10 统计（柱状图）。
*   **周转与故障率**：近一年内资产采购与维修费用趋势图（折线图）。

## 4. 非功能性需求与高可用要求

1.  **高并发数据一致性**：大量并发领用相同热门资产（如公共测试机/投影仪）时，必须通过分布式锁/乐观锁保证库存绝对不会超发。
2.  **安全性设计**：
    *   **数据隔离**：数据级权限控制。不同部门的管理者默认仅能查看本部门及下属部门的资产数据；高管或特定角色可看全局。
    *   **接口防刷与鉴权**：所有业务接口必须携带 Token 且进行鉴权拦截，关键操作防止越权访问 (IDOR)。
3.  **系统可追踪性（审计）**：
    *   对 `ams_asset` 和流程表的核心修改动作，必须自动记录操作日志表（操作人、时间、变更前/变更后 JSON），供审计回溯。
4.  **易用性与体验**：
    *   中后台提供极致的响应速度。图片、合同照片等大文件通过独立的分布式对象存储 (OSS) 处理，不阻塞业务应用带宽。
