# 企业资产管理系统 (EAM) 开发计划表

基于最新的《产品需求文档 (PRD)》与《系统设计方案》，我为您制定了以下的分阶段敏捷开发计划。整个项目周期预计为 **6 - 8 周**，按功能模块和技术底座进行递进式交付。

## 第一阶段：基建与底层架构 (Week 1)
**目标**：完成系统前后端工程初始化、核心组件搭建与数据库结构设计。

*   **环境与工程初始化 (2天)**：
    *   搭建微服务构建框架 (Spring Boot 3 + JDK 17) 及前端 (React 18 + Vite) 骨架。
    *   配置 Git 仓库规范、Docker Compose 基础容器环境（MySQL, Redis, MinIO, Flowable）。
*   **权限与基础 RBAC (3天)**：
    *   开发 `sys_user`, `sys_role`, `sys_dept` 基础模块。
    *   实现基于 JWT 的登录认证与接口拦截。
    *   **核心攻关**：配合复杂的部门与“中心主任、部门长”等多级负责人标识，完成组织架构树的树形查改能力。

## 第二阶段：核心资产台账业务 (Week 2 - Week 3)
**目标**：实现资产的 CRUD 以及基础卡片全生命周期时间轴。

*   **资产分类与台账 (4天)**：
    *   实现字典/分类管理（资产大类、小类）。
    *   开发核心 `ams_asset` 模块（手动录入、条件分页检索）。
    *   完成条码/二维码的自动生成与批量后端下载接口。
*   **批量操作与履历追溯 (3天)**：
    *   开发 Excel 批量导入资产功能。
    *   开发 `ams_asset_record` 履历聚合查询，构建资产全生命周期时间轴前端 UI 展示。
*   **移动端扫码基座 (3天)**：
    *   搭建移动端 H5 或小程序基础工程。
    *   实现扫码获取资产详情与状态的功能。

## 第三阶段：工作流引擎与流转业务 (Week 4 - Week 5)
**目标**：接入 BPMN，实现资产流转所有的线上审批。

*   **流程引擎深度集成 (4天)**：
    *   集成 Flowable 7.0，实现画板接入与模型部署。
    *   打通引擎与 `sys_dept` 组织架构，实现基于动态表达式的审批寻址（获取部门长、中心主任）。
*   **核心业务单据流转 (6天)**：
    *   实现 **领用、退库** 业务线的表单发起、流程审批与台账状态联动更新。
    *   实现复杂的 **跨部门调拨** （包含多方审批验证机制）。
    *   开发 **报修与报废** 模块（包含财务审核残值的流转设定）。

## 第四阶段：高可用外围设施与看板验收 (Week 6 - Week 7)
**目标**：完善并发处理、运维指标、高级数据可视化并进入联调测试。

*   **数据看板与统计报表 (3天)**：
    *   使用 ECharts 开发多维数据看板（资产分布、折旧状态、部门利用率）。
*   **并发扩展与运维能力 (3天)**：
    *   实装 Redis 分布式锁，重点验证“公共热门资产领用防超发”机制。
    *   打通 MinIO 存储接口，实现流程中的合同、报修图片上传业务。
*   **联调测试与 UAT (4天)**：
    *   前后端全链路联合调试，压力测试。
    *   进行 UAT (用户验收测试)，修复 Bug，形成用户操作手册。
