# 任务执行梳理与后续计划

## 1. 「Workflow Task Verification」任务现状梳理

### ✅ 已完成内容
1. **工作流后端基建**:
   - 集成 Flowable 7.0 引擎，封装 `FlowableService` 通用操作。
   - 实现 `WorkflowTaskVO` 业务视图对象，支持任务列表与业务单据（领用原因、资产名称等）的关联展示。
   - `DeptRoutingService` 实现动态寻址逻辑（自动查找部门长、中心主任）。

2. **资产领用流程 (AssetBorrowProcess)**:
   - 完成 BPMN 流程定义：`部门负责人` -> `中心主任` -> `管理员确认发放`。
   - 实现 `AssetStateExecutionListener`：流程结束自动将资产状态更新为“领用中”。
   - 业务逻辑闭环：`AmsRequestBillServiceImpl` 统一处理单据保存、流程启动与变量注入。

3. **前端对接**:
   - `src/api/workflow.ts` 定义了完整的提交、查询、审批接口。
   - `src/pages/workflow/tasks` 页面骨架已搭建，支持查看待办任务列表。

### 🔍 待优化/已知问题
- **审批详情页**: ✅ 已实现 `BillDetail` 并在审批弹窗中集成。
- **流程弹性**: ✅ 已在 BPMN 中加入驳回路径。
- **状态同步粒度**: ✅ 已实现驳回监听器。

## 2. 后续开发计划 (Phase 3 补完 & Phase 4 规划)

### 第一阶段：流程闭环与用户体验增强 (预计 1-2 周)
#### [后端]
- [ ] **流程完善**: 在 BPMN 中加入“驳回”网关逻辑，支持 `approved=false` 时跳转起始节点。
- [ ] **多流程支持**: 复制并调整 `AssetBorrowProcess` 为 `AssetReturnProcess` (退库) 和 `AssetTransferProcess` (调拨)。
- [ ] **消息通知**: 集成简单的站内信（或模拟日志），在任务到达时通知办理人。

#### [前端]
- [ ] **审批详情组件**: 开发通用的 `BillDetail` 组件，根据 `billType` 渲染不同的表单镜像。
- [ ] **我的申请中心**: 完善“我的申请”页面，支持查看流程进度（进度条/步骤条）。

### 第一阶段：高级业务流程实装 (Phase 4)
#### [后端]
- [ ] **报修流程 (AssetRepairProcess)**: 
  - 节点：`部门报备` -> `IT核损` -> `外部维修/更换`。
  - 效果：审批中标记资产为“报修中”，防止误领用。
- [ ] **报废流程 (AssetScrapProcess)**:
  - 节点：`中心主任` -> `资产管理员` -> `财务确认`。
  - 效果：审批后资产进入“已报废”库，不再出现在活跃台账。
- [ ] **生命周期历程**: 提供 `asset/history/{assetId}` 接口，返回该资产的所有流转单据（领用、退库记录）。

#### [前端]
- [ ] **资产选择器**: 开发 `AssetSelect` 弹窗，取代手动输入资产 ID 的落后交互。
- [ ] **生命周期时间轴**: 在资产台账点击详情，展示从“购入”到“当前位置”的整条 Timeline。

### 第二阶段：数据决策与盘点 (Phase 5)
- [ ] **可视化看板**: 首页集成 Echarts，展示资产净值分布、分类比例、部门占用热力图。
- [ ] **预警系统**: 维保过期提醒、报废年限自动探测。
- [ ] **电子盘点**: 支持生成盘点任务单，手机端扫码对碰。

## 3. 风险点与技术决策
- **技术决策**: 流程变量中保持存储 `initiatorId` 和 `billId`，确保历史回溯。
- **性能**: 目前任务列表采用遍历关联业务表，后续任务量大时建议优化为 SQL Join 或冗余 business_info 至流程表。
